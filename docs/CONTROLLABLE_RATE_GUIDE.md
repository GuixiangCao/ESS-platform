# 可控收益率功能说明

## 功能概述

可控收益率是一个更准确的收益评估指标，用于评估电站在实际运营天数（排除无收益天数）的表现。

## 计算方法

### 基本原理

可控收益率 = (可控实际收益 / 可控预期收益) × 100%

其中：
- **可控实际收益** = 排除实际收益为0的天数后的实际收益总和
- **可控预期收益** = 排除实际收益为0的天数后的预期收益总和

### 计算示例

假设某电站9月份的数据如下：

| 日期 | 实际收益 | 预期收益 |
|------|---------|---------|
| 9.1  | 1000    | 1200    |
| 9.2  | 1100    | 1200    |
| ... | ...     | ...     |
| 9.20 | 0       | 1200    |
| 9.21 | 0       | 1200    |
| ... | ...     | ...     |
| 9.30 | 1000    | 1200    |

**传统收益达成率计算**：
```
达成率 = (整个9月实际收益) / (整个9月预期收益) × 100%
```

**可控收益率计算**：
```
可控收益率 = (整个9月实际收益 - (9.20的实际收益 + 9.21的实际收益))
           / (整个9月预期收益 - (9.20的预期收益 + 9.21的预期收益)) × 100%

即：排除9.20和9.21这两天没有充放电的数据后计算的达成率
```

## 为什么需要可控收益率？

### 传统达成率的局限性

传统的收益达成率会将所有天数包括在内，包括：
- 计划性停运日
- 设备检修日
- 极端天气导致的停运日
- 其他不可控因素导致的无收益日

这些天数的收益为0，但预期收益仍然计入，会拉低整体达成率。

### 可控收益率的优势

1. **更真实的运营表现**：只评估电站实际运营天数的表现
2. **排除不可控因素**：自动排除无收益日的影响
3. **更精准的评估**：反映电站在正常运营状态下的真实效率
4. **便于对比分析**：不同电站之间的停运天数不同，可控收益率更具可比性

## 数据展示

### 电站列表页面

在电站列表页面，"收益达成率"后面新增了"可控收益率"列：

```
| 电站编号 | 电站名称 | ... | 收益达成率 | 可控收益率 | ... |
|---------|---------|-----|-----------|-----------|-----|
| 1       | 电站A   | ... | 85.50%    | 92.30%    | ... |
| 2       | 电站B   | ... | 78.20%    | 88.10%    | ... |
```

### 样式说明

- **收益达成率**：使用绿色/黄色/红色表示优秀/良好/需改进
- **可控收益率**：使用蓝紫色系表示，视觉上与达成率区分
  - 优秀 (≥90%): 蓝色
  - 良好 (70%-90%): 紫色
  - 需改进 (<70%): 深紫色

### 汇总行

表格底部的汇总行会显示：
- 总收益达成率
- 总可控收益率

## 技术实现

### 后端实现

在 MongoDB 聚合查询中使用条件求和：

```javascript
controllableExpected: {
  $sum: {
    $cond: [
      { $gt: ['$actualRevenue', 0] },  // 如果实际收益 > 0
      '$expectedRevenue',               // 则累加预期收益
      0                                 // 否则加0
    ]
  }
}
```

### 前端显示

新增字段：
- `controllableExpected`: 可控预期收益
- `controllableActual`: 可控实际收益
- `controllableRate`: 可控收益率
- `activeDays`: 有效运营天数

## 使用场景

### 1. 电站性能评估

使用可控收益率评估电站在实际运营状态下的性能，排除计划停运的影响。

### 2. 运维质量分析

对比达成率和可控收益率的差异：
- **差异小**：说明停运天数少，运维质量好
- **差异大**：说明停运天数多，需要优化运维计划

### 3. 设备状态监控

可控收益率高但达成率低，可能表示：
- 频繁的计划性维护
- 设备故障导致多次停运
- 需要优化维护计划

### 4. AI vs 普通电站对比

更公平地对比AI电站和普通电站的表现，排除不同停运策略的影响。

## 注意事项

1. **数据准确性**：确保实际收益为0的数据是真实的停运日
2. **阈值设置**：实际收益 > 0 作为有效运营日的判断标准
3. **极小值处理**：实际收益很小但不为0的日期仍计入可控收益
4. **数据完整性**：需要完整的历史数据才能准确计算
